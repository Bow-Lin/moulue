# v0.2 Profile-driven Multi-character Chat

## Goal

Upgrade the system from a single-character chat MVP to a profile-driven multi-character runtime.

## Completed

- convention-based profile loading: `agentId -> profiles/{agentId}.yaml`
- built-in profiles:
  - `zhuge_liang`
  - `cao_cao`
- Profile Schema v1
- structured prompt assembly
- CLI agent selection:
  - `bun run chat --agent cao_cao`
  - `bun run chat -a cao_cao`
- HTTP agent selection:
  - `POST /chat` with `agentId`
- memory isolation by `user_id + agent_id`
- debug trace:
  - `beforeReply`
  - `afterReply.extraction`
- tests pass

## Out of Scope

- multi-agent collaboration
- world state
- scenario engine
- frontend
- vector search
