# Chat CLI Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a temporary terminal chat client that reuses the local `POST /chat` endpoint and automatically discovers the active dev server port.

**Architecture:** Keep the CLI as a thin client over the existing HTTP API. The dev server writes its active port to a runtime metadata file, and the CLI reads that file to find the correct local server before sending chat requests.

**Tech Stack:** Bun, TypeScript, Bun HTTP server, Node/Bun readline

---

### Task 1: Add runtime metadata writing for the dev server

**Files:**
- Create: `src/app/runtime-file.ts`
- Modify: `src/app/server.ts`
- Modify: `.gitignore`

**Step 1: Write the runtime file helper**

Implement helpers to:

- ensure `/.runtime/` exists
- write `dev-server.json`
- read `dev-server.json`

**Step 2: Wire the dev server**

After `Bun.serve()` succeeds, write the active server port to `/.runtime/dev-server.json`.

**Step 3: Verify manually**

Run: `bun run dev`
Expected: server starts and `/.runtime/dev-server.json` is created with the active port

### Task 2: Add the terminal chat client

**Files:**
- Create: `src/app/chat-cli.ts`
- Modify: `package.json`

**Step 1: Implement the CLI**

Implement a terminal loop that:

- reads the server port from `/.runtime/dev-server.json`
- calls `POST /chat`
- prints replies
- keeps the returned `sessionId`
- exits on `quit` or `exit`

**Step 2: Add the script**

Add:

```json
"chat": "bun run src/app/chat-cli.ts"
```

**Step 3: Verify manually**

Run: `printf '如何治国？\nquit\n' | bun run chat`
Expected: one real chat reply is printed and the command exits cleanly
